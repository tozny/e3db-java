Class QueryParams

  • Object
    • com.tozny.e3db.QueryParams

  • public class QueryParams
extends Object
    Holds all parameters for an E3DB query operation.

    This class allows you to specify all parameters for performing a query against E3DB (using the Client.query(QueryParams, ResultHandler) method). Instances of this class must be created using the QueryParamsBuilder class.

    By default, queries are limited to 50 records, only return records written by the client, and results do not include the data associated with each record (only the Record.meta() portion). See the QueryParamsBuilder class for documentation about configuring these parameters.

    Given an instance of this class, you can use the buildOn() method to create a QueryParamsBuilder that is pre-configured to the values held by the QueryParams instance. This makes pagination easy to implement, where the after parameter is the only thing that needs to vary.

    Behavior with Multiple Filters

    All filters specified act as conjunctions; that is, a record must match every filter criteria to be returned. For example, if record types and writer IDs are given (via types and writerIds), then records will only be returned if they were written by one of the given IDs and if they have the given record type.

    Getting all Shared Records

    When true, the includeAllWriters flag indicates that results should include all records shared with the client making the query request.

    Filtering Writers

    If writerIds is also set, then results will be limited to records written by the set of IDs. If the calling client does not include their own ID in the set, their records will not be included in the results!

    • Field Detail

      • count

        public final int count
        The maximum number of records to return. If -1, uses the server default of 50.

      • includeData

        public final Boolean includeData
        If true, include record data with results. Otherwise, only include metadata. Defaults to null, which uses the server default of false.

        When this value is true, then Record.data() will contain the record's unencrypted data. If false or null, data() will return an empty Map.

        In either case, Record.meta() will always return metadata about the record.

      • writerIds

        public final List<UUID> writerIds
        If not null, limit results to records written by the given set of IDs. If empty, no records will be returned. Defaults to null.

      • userIds

        public final List<UUID> userIds
        If not null, limit results to records about given set of IDs. If empty, no records will be returned. Defaults to null.

      • recordIds

        public final List<UUID> recordIds
        If not null, limit results to the records specified. If empty, no records will be returned. Defaults to null.

      • types

        public final List<String> types
        If not null, limit results to records of the given types. If empty, no records will be returned. Defaults to null.

      • after

        public final long after
        If greater than 0, limit results to the records appearing "after" the given index. Defaults to 0, which specifies that results start with the first record that matched.

        For use with QueryResponse.last() when implementing pagination.

      • includeAllWriters

        public final Boolean includeAllWriters
        If true, include all records shared with this client. Defaults to null, which uses the server default of false.

        Note that writerIds must be null when this is true, or writerIds will take precedence.
